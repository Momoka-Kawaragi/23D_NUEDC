## CubeMX 配置

一共需要配置13个引脚，任意空闲普通io都可以

```c
extern driverIO SDIO0;
extern driverIO SDIO1;
extern driverIO SDIO2;
extern driverIO SDIO3;
extern driverIO PDC;
extern driverIO RST;
extern driverIO SCLK;
extern driverIO CS;
extern driverIO UPDATE;

extern driverIO PS1;
extern driverIO PS2;
extern driverIO PS3;
```

需要在CubeMX里面将GPIO设置为GPIO_OUTPUT模式，然后自定义IO名称为上述的**SDIO0**之类的

AD9959驱动使用示例   

## 外部接口

| 函数 | 说明 |
|------|------|
| `void Init_AD9959(void)` | 初始化AD9959（PLL配置、四通道默认参数） |
| `void Write_Frequence(uint8_t Channel, uint32_t Freq)` | 设置通道频率，`Channel` 取值 0~3，`Freq` 单位 Hz（0~50MHz） |
| `void Write_Amplitude(uint8_t Channel, uint16_t Ampli)` | 设置通道幅值，`Ampli` 取值 0~1023 |
| `void Write_Phase(uint8_t Channel, uint16_t Phase)` | 设置通道相位，`Phase` 取值 0~16383 对应 0~360° |
| `void AD9959_IO_Update(void)` | 发送 IO UPDATE 脉冲，设置完参数后必须调用才生效 |

## 使用示例

初始化AD9959，设置通道2输出正弦信号（增益1023、相位0、频率50kHz）：

```c
Init_AD9959();
Write_Frequence(2, 50000);
Write_Amplitude(2, 1023);
Write_Phase(2, 0);
AD9959_IO_Update();  // 设置完参数必须调用此函数才会生效
```

## Profile 引脚 (PS0~PS3) 
## 是什么

AD9959 每个通道有 **16 套 profile**（Profile 0~15），每套可以预设不同的频率/相位/幅度。PS0~PS3 4 个引脚的电平组合决定当前用哪一套：

```
PS3 PS2 PS1 PS0  →  选中 Profile  →  使用的寄存器
 0   0   0   0   →  Profile 0     →  CFTW0 (频率) + CPOW0 (相位) + ACR (幅度)
 0   0   0   1   →  Profile 1     →  CW1   (频率/相位/幅度)
 0   0   1   0   →  Profile 2     →  CW2
 0   0   1   1   →  Profile 3     →  CW3
 0   1   0   0   →  Profile 4     →  CW4
 ...                                (AD9959 只到 CW4，共 5 个额外 profile)
```

> AD9959 的 CW1~CW4 对应 4 个额外 profile，加上 Profile 0 共 5 个。4 个 PS 引脚最多选 16 个，但芯片实际只支持到 CW4。

### 为什么要用 Profile

切换 profile 是**纯硬件行为**——改变 PS 引脚电平后，AD9959 在 **几纳秒内** 直接切到另一套参数，不需要 SPI 通信。对比 SPI 改写寄存器的方式：

| | Profile 切换 (PS 引脚) | SPI 改写寄存器 |
|---|---|---|
| 切换耗时 | ~几个 ns | ~5-10 µs |
| 最高调制速率 | MHz 级 | 受 SPI 速度限制 (~100 kbps) |
| 时序抖动 | 几乎为零 | SPI 传输长度波动 |
| CPU 开销 | 一个 GPIO 翻转 | 整段 SPI 传输 |

### 调制中的应用

**2FSK / 2ASK** 只用一个 PS0（2 个状态 = 2 个 profile），其余 PS1~PS3 拉低不用。

```
2FSK:  SET_2FSK(0, 100k, 10M)
       ┌──────────────────────────────────┐
       │ Profile 0 (PS0=0): 频率 = 100 kHz │
       │ Profile 1 (PS0=1): 频率 = 10 MHz  │
       └──────────────────────────────────┘

2ASK:  SET_2ASK(0, 60M, 100, 1000)
       ┌───────────────────────────────────┐
       │ Profile 0 (PS0=0): 60MHz, 幅度=100 │
       │ Profile 1 (PS0=1): 60MHz, 幅度=1000│
       └───────────────────────────────────┘
```

### 扩展：多级调制

同时用 PS0 + PS1（4 个组合）可以实现 **4FSK**——2 bit 数据一次发出：

```
PS1 PS0
 0   0  →  Profile 0: 频率 f00  (数据 00)
 0   1  →  Profile 1: 频率 f01  (数据 01)
 1   0  →  Profile 2: 频率 f10  (数据 10)
 1   1  →  Profile 3: 频率 f11  (数据 11)
```

### CubeMX 配置

4 个 PS 引脚都设为 GPIO_Output，初始化时全部拉低（`InitIO_9959()` 已处理）。

```c
WRT(PS0, 0);  // InitIO_9959() 中，全部初始化为低电平
WRT(PS1, 0);
WRT(PS2, 0);
WRT(PS3, 0);
```

## CMake 工程配置

```cmake
# 添加驱动源文件
target_sources(${CMAKE_PROJECT_NAME} PRIVATE
    MY_DRIVE/MY_DDS/ad9959.c
)

# 添加驱动头文件路径
target_include_directories(${CMAKE_PROJECT_NAME} PRIVATE
    MY_DRIVE/MY_DDS
)
```
