# TEA5767 FM 收音模块 — 软件 bit-bang I2C 驱动

> 适用芯片: **NXP TEA5767** (FM 收音, 76 ~ 108 MHz)
> 仓库路径: `Core/MY_DSP/MY_TEA5767/tea5767.{c,h}`
> 参考: 原工程 `board_tea5767.c` (HAL_I2C1), 本 driver 用软件 bit-bang I2C 替代, 复用其寄存器算法 / 搜台策略

---

## 1. 硬件信号链

```
┌──────────┐  FM (76-108 MHz)  ┌──────────┐  LOUT/ROUT  ┌──────────┐  喇叭
│  天线    │ ───────────────► │  TEA5767 │ ──────────► │  功放    │ ────►
└──────────┘                  └──────────┘             └──────────┘
   (75Ω 阻抗匹配)               │ 32.768 kHz 晶振
                                │ I2C (SCL / SDA)  0x60
                                ▼
                            MCU STM32F407
```

- **天线**: 75 Ω 输入, FM 频段; 卖家一般自带 50 cm 拉杆, 短线需配同轴转 SMA
- **音频**: LOUT / ROUT 直接接功放 (LM386 / TPA2005 / MAX98357A 等), **本 driver 不管音频通路**
- **I2C**: 3.3V, 软件 bit-bang ~100 kHz, 地址 **0x60**

---

## 2. CubeMX 引脚配置 (User Label)

| 任一空闲 GPIO (开漏) | `TEA5767_SCL` | I2C SCL (串行时钟) |
| 任一空闲 GPIO (开漏) | `TEA5767_SDA` | I2C SDA (双向数据) |

**注意事项**:
1. **SCL / SDA 必须配成 `GPIO_OUTPUT_OD` (开漏)** — driver 内部虽然会强制再设一次, 但 CubeMX 默认 `OUTPUT_PP` 在同一总线上和别的推挽设备会撞
2. **总线上必须外接上拉电阻 (4.7 kΩ → 3.3V)**, SCL 和 SDA 都要
3. 避开 `PB10 / PB11` — 已被 SI5351 (I2C2) 占用, 共享总线会冲突
4. **避开 SWD 调试脚 `PA13 / PA14`** — CubeMX 默认占用, 改了会丢调试
5. 推荐使用: `PB0` / `PB1` (如果空), 或 `PC12` / `PD2`

设置方法: CubeMX → Pinout view → 选 GPIO → 设 `GPIO_Output` + User Label `TEA5767_SCL`(同理 SDA) → GENERATE CODE, `main.h` 会自动展开:

```c
#define TEA5767_SCL_Pin GPIO_PIN_xx
#define TEA5767_SCL_GPIO_Port GPIOxx
#define TEA5767_SDA_Pin ...
#define TEA5767_SDA_GPIO_Port ...
```

## 3. 核心 API

```c
#include "tea5767.h"

TEA5767_Init();                       // 初始化 + 设默认频点 88.0 MHz
TEA5767_SetFrequency(98700);          // 98.7 MHz (kHz 单位)
TEA5767_Search(TEA5767_SEARCH_UP);    // 手动 +100 kHz
TEA5767_AutoSearch(TEA5767_SEARCH_UP); // 自动搜台 (带 RF/IF 锁定检测)
TEA5767_Mute(TEA5767_MUTE_ON);        // 静音 / 解除
TEA5767_SetAudioMode(TEA5767_AUDIO_MONO); // 强制单声道 (去除立体噪声)

/* 扫频结果 (main loop 周期调用) */
TEA5767_Status_t s;
TEA5767_GetStatus(&s);
printf("f=%u kHz RF=%d ST=%d IF=0x%02X LEV=%d\n",
       s.frequency_khz, s.rf, s.stereo, s.if_counter, s.level_adc);

if (TEA5767_IsLocked())  { /* 锁定 */ }
if (TEA5767_IsStereo())  { /* 立体声 */ }
uint8_t rssi = TEA5767_GetRSSI();     // 0~15
uint8_t ifc  = TEA5767_GetIFCounter(); // 0x31~0x3E 为锁定
```

## 4. 5 字节写寄存器格式

| Byte | bit 7 | bit 6 | bit 5 | bit 4 | bit 3 | bit 2 | bit 1 | bit 0 | 默认 |
| :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: |
| **0** | MUTE | SM | PLL[13:8] (6 bit) | | | | | | 0x31 |
| **1** | PLL[7:0] | | | | | | | | 0xA0 |
| **2** | SUD | SSL[2:0] | | | HLSI | MS | MR | ML | 0x20 |
| **3** | PLLREF | DTC | 0 | 0 | mono_en | SNC | SI | 0 | 0x11 |
| **4** | 0 | 0 | 0 | 0 | 0 | 0 | 0 | 0 | 0x00 |

- **MUTE**: 1 = 静音
- **SM**: 1 = 进入搜索模式 (驱动会调谐停止)
- **PLL[13:0]**: 频率合成字, 算法见 §6
- **SUD**: 1 = 向上搜台, 0 = 向下
- **SSL[2:0]**: 搜索停止电平 (7 = 最低灵敏度, 0 = 最高)
- **HLSI**: 1 = 高边注入, 0 = 低边注入 (决定 §6 加减 225 kHz)
- **PLLREF**: 0 = 32.768 kHz, 1 = 6.5 MHz (外灌)
- **DTC**: 去加重时间常数 (0 = 75 µs, 1 = 50 µs)
- **mono_en**: 1 = 强制单声道
- **SNC**: 立体声噪声消除

## 5. 5 字节读寄存器格式 (状态)

| Byte | bit 7 | bit 6 | bit 5 | bit 4 | bit 3 | bit 2 | bit 1 | bit 0 |
| :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: | :--: |
| **0** | RF | BLF | PLL[13:8] | | | | | |
| **1** | PLL[7:0] | | | | | | | |
| **2** | STEREO | IF[6:0] | | | | | | |
| **3** | LEV[3:0] | | | | 0 | | | |
| **4** | CI | 0 | | | | | | |

- **RF**: 1 = 已锁台 (Ready)
- **BLF**: 1 = 带宽极限标志 (频偏太大)
- **STEREO**: 1 = 立体声接收
- **IF[6:0]**: 中频计数, 锁定范围 `0x31~0x3E`
- **LEV[3:0]**: 场强 ADC, 0~15, 越大信号越强
- **CI**: 1 = 芯片 ID 错误 (异常)

## 6. PLL 算法 (与参考 driver 一致)

**写频** (kHz → 14-bit PLL):
```
PLL = 4 * (F ± 225) / 32.768      (整数部分)
  其中 ± 取决于 HLSI 位
```

**读频** (PLL → kHz):
```
F = PLL * 8.192 ∓ 225
```

> **为什么 ±225**: TEA5767 的中频是 225 kHz, HLSI 决定本振频率相对于信号是偏高还是偏低 225 kHz

## 7. 软件 bit-bang I2C 时序

```
SCL: _____|‾‾‾‾|_____ × 8 次 + ACK ─────|‾‾‾‾|_____
SDA: ‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾‾
     ▲ start 写字节 0  写字节 1 ... 写字节 4 ▲ stop
```

- SCL 高 / 低 各 ~5 us, 总线 ~100 kHz (跟 TEA5767 时序兼容, 上限 400 kHz)
- 写: 主 → 从 (8 bit MSB first + 第 9 clock 从回 ACK)
- 读: 从 → 主 (8 bit MSB first + 第 9 clock 主回 ACK/NACK, 最后一字节发 NACK)
- **没有错误重试**: 一次失败就报 `read NACK`, 上层 (`AutoSearch`) 会 retry

## 8. main.c 调用示例

```c
#include "tea5767.h"

int main(void) {
    HAL_Init();
    SystemClock_Config();
    MX_GPIO_Init();
    /* 其它 CubeMX 初始化 */

    TEA5767_Init();                              // 默认 88.0 MHz

    /* 选一个 demo: 手动搜台 / 自动搜台 / 直接定频 */
    TEA5767_SetFrequency(98700);                 // 98.7 MHz (中央人民)
    // TEA5767_AutoSearch(TEA5767_SEARCH_UP);    // 自动向上搜台

    while (1) {
        TEA5767_Status_t s;
        TEA5767_GetStatus(&s);                  // 每次 I2C 读 5 字节, ~30 ms 后回填
        if (s.ready) {
            printf("[FM] f=%u kHz RF=%d ST=%d IF=0x%02X LEV=%d\n",
                   s.frequency_khz, s.rf, s.stereo, s.if_counter, s.level_adc);
        }
        HAL_Delay(500);                          // 不要太快, 避免 I2C 总线占用
    }
}
```

## 9. 已知坑 / 排查

| 现象 | 原因 | 解决 |
| :--- | :--- | :--- |
| 串口一直打 `TEA5767: write NACK` | I2C 没应答 | 查: (1) SCL/SDA 是否开漏 + 上拉; (2) CubeMX User Label 是否生效; (3) 总线是否被其它设备拉住 |
| `AutoSearch` 走完整个 76~108 一无所获 | (1) 天线没接 / 接触不良; (2) 灵敏度低 (SSL=7) | 加长天线 / 把搜索写寄存器 `byte2 = 0x80` (SSL=0, 最高灵敏度) |
| 调谐到 88.0 MHz 听不到声音 | (1) 音频 LOUT/ROUT 没接功放; (2) 静音打开 | 查: `TEA5767_Mute(TEA5767_MUTE_OFF)`; 用耳机或示波器测 LOUT |
| `RF=0` 但 `IF` 在 `0x31~0x3E` | 锁定 IF 但场强太低 | 这是真·弱信号, 不是 bug |
| `RF=1` 但串口听不到 `AutoSearch OK` | 锁定但停在另一条指令 | 看完整 log, 一般是 `IF` 出范围 |

## 10. 与参考 driver (`board_tea5767.c`) 的差异

| 维度 | 参考 board_tea5767 | 本 driver |
| :--- | :--- | :--- |
| I2C | `HAL_I2C1_Master_Transmit` | **软件 bit-bang** (CubeMX User Label 引脚) |
| I2C 句柄 | `&hi2c1` (工程未启用, 编译会 fail) | **不需要** I2C 外设 |
| 引脚 | 不写 | `TEA5767_SCL` / `TEA5767_SDA` (可换板子) |
| PLL 算法 | 一致 | 一致 |
| 搜台策略 | 一致 | 一致 |
| `GetStatus` | 无 (需自己解析) | **新增** (一次性填好, 便于 main loop) |
| `IsLocked` / `IsStereo` / `GetRSSI` | 无 | **新增** |
| API 名字 | 一致 (`TEA5767_Init` / `SetFrequency` / `Search` / `AutoSearch` / `Mute` / `SetAudioMode`) | **完全一致** |

> **移植注意**: 本 driver 替换参考 `board_tea5767.c` 时, `main.c` 的 `#include "board_tea5767.h"` 改成 `#include "tea5767.h"`, 函数名 / 行为不变。
